Highlight #1 What is Version Control?

Version Control System (VCS) is a system that records changes to of file or set of files over time so that you can recall specific version of file later. It also allow to revert files, keep track history of changes or record or archives of any activities to the files.

Highlight #2 Distributed Version Control (DVC)

Distributed Version Control work with several remote repository system and collaborating with differetn groups of people in different ways and simultaneously with in the same project.This is the main benefit. it improve efficiency in work and organization.

Highlight #3 VCS - Git Basics

It introduces the important of version control systems and tracking changes made to sources code. It automatically make notes to the file that working on. It keep version of history and merge changes to the same files. Git has three main states that a files can reside in. These three states are committed, modified, and staged.

Highlight #4 Git Basis Benefits

Git allows groups of people to work on the same documents after code at the same time and without interfere with each other tasks that be working on.

Highlight #5 VCS Benefits

Files can not be overwritten. There is common repository that holds all the latest files.It allows one to revert back to an older version of the file or project if needed. Version Control system will back up files so that we don't loose work or have to start a project over.
